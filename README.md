# get_brms_recovery_report

Downloads the IBM i BRMS Recovery Report spool file (**QP1ARCY**) via SSH — no IBM i Navigator or REST API required. Works from any Linux, macOS, or Windows machine with SSH access to the IBM i.

Both scripts follow the same three-step flow:

1. **Locate** the most recent `QP1ARCY` spool file for the user via `QSYS2.OUTPUT_QUEUE_ENTRIES` SQL
2. **Copy** it to an IFS temp file using `CPYSPLF`
3. **Download** it via SCP and delete the temp file

---

## Prerequisites

### IBM i side

| Requirement | How to verify |
|---|---|
| IBM i V7R5 or later | `DSPPTF` |
| SSH server running | `WRKACTJOB SBS(QSYSWRK)` → look for `SSHD` |
| PASE installed (5770-SS1 Option 33) | `GO LICPGM` |
| `db2` CLI available in PASE | `ls /QOpenSys/usr/bin/db2` on IBM i |
| User profile authorised to `CPYSPLF` target spool files | `WRKOBJPDM QBRMS *OUTQ` |

### Client side

| Script | Requirements |
|---|---|
| `get_qp1arcy.sh` | Bash, `ssh`, `scp`, `jq` |
| `Get-QP1ARCY.ps1` | PowerShell 5.1+ (Windows) or PowerShell 7+ (cross-platform), [Posh-SSH](https://github.com/darkoperator/Posh-SSH) module |

## Setup

### 1. Generate an SSH key pair (one-time)

```bash
ssh-keygen -t rsa -b 4096 -f ~/.ssh/ibmi_id_rsa
```

### 2. Install the public key on the IBM i

```bash
ssh-copy-id -i ~/.ssh/ibmi_id_rsa.pub IBMIUSER@192.168.10.50
```

If `ssh-copy-id` is not available, append `ibmi_id_rsa.pub` contents to `~/.ssh/authorized_keys` on the IBM i manually.

### 3. Configure credentials

```bash
cp ibmiscrt.json.template ibmiscrt.json
```

Edit `ibmiscrt.json` with your IBM i username and the path to your private key:

```json
{
  "user": "IBMIUSER",
  "key":  "/home/user/.ssh/ibmi_id_rsa"
}
```

> `ibmiscrt.json` is listed in `.gitignore` and will never be committed.

### 4. Make the Bash script executable

```bash
chmod +x get_qp1arcy.sh
```

### 5. Install Posh-SSH (PowerShell only)

```powershell
Install-Module -Name Posh-SSH -Scope CurrentUser
```

## Usage

### Bash

```bash
./get_qp1arcy.sh 192.168.10.50
```

### PowerShell

```powershell
.\Get-QP1ARCY.ps1 192.168.10.50
```

If your execution policy blocks unsigned scripts on Windows:

```powershell
Set-ExecutionPolicy -Scope Process -ExecutionPolicy Bypass
.\Get-QP1ARCY.ps1 192.168.10.50
```

### Output

Both scripts save the report to the current directory as:

```
QP1ARCY_YYYYMMDD_HHMMSS.txt
```

## Notes

**SSH key auth:** Both scripts require key-based SSH authentication. Password-based auth is intentionally not supported. The private key path is read from `ibmiscrt.json` at runtime.

**CPYSPLF encoding:** The `WSCST(*AUTOCVT)` parameter handles EBCDIC-to-ASCII conversion automatically, so the downloaded `.txt` file is immediately readable on the client.

**IBM i PASE db2 CLI:** The SQL query runs via `/QOpenSys/usr/bin/db2`, which is part of the IBM i base OS on V7R5. If it is not present, verify that PASE (5770-SS1 Option 33) is installed. The script checks for the binary before running the query and exits with a clear error if it is missing.

**SQL view:** The query uses `QSYS2.SPOOL_INFO` (available since V7R1) with columns `FILE_NUMBER` and `CREATE_TIMESTAMP`. Do not confuse this with `QSYS2.OUTPUT_QUEUE_ENTRIES`, which uses different column names and may be a table function depending on PTF level.

**Spool file scope:** The query filters by `JOB_USER = <your user>`. If the `QP1ARCY` spool is generated by a different job user (e.g. a BRMS service account), update the `WHERE` clause in the SQL inside the script.

---

## IBM i Compatibility

`get_qp1arcy.sh` runs on Linux/macOS and connects to IBM i over SSH. The remote commands it executes (`db2`, `system`, `rm`) use IBM i PASE paths and avoid any Linux-only constructs, making them safe to run in the IBM i PASE shell if needed.
